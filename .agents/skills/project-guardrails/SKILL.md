Loop Task App is an **Electron desktop app** (main + preload + renderer) written
in **strict TypeScript** with **React 19**. It monitors loop-task daemons over
HTTP + SSE. It is **read-only** in v1.

## Architecture Constraints

- **[enforced] All network I/O lives in the main process** (`src/main/`). The
  renderer must NEVER call `fetch`/HTTP/SSE directly ā€” the loop-task daemon
  sends no CORS headers and the renderer is sandboxed. Add network work to the
  main process and expose it via IPC.
- **[enforced] The renderer reaches the outside world only through `window.api`**
  (the `contextBridge` in `src/preload/index.ts`). Never expose `ipcRenderer`
  (or any Node/Electron primitive) directly to the renderer.
- **[enforced] The IPC contract is the single source of truth:**
  `src/shared/ipc.ts`. Any new IPC channel MUST be typed there and consumed by
  main, preload, and renderer through those types. Keep the surface minimal.
- **[enforced] Respect layer boundaries:** `src/renderer/**` must not import from
  `src/main/**` or Node built-ins; `src/main/**` and `src/preload/**` must not
  import renderer/React code. Only `src/shared/**` is shared by all layers.
- **[enforced] Poll loops, push logs.** The loop list is polled (5s) because the
  daemon has no server-fed event stream; logs use SSE. Do not add ad-hoc
  polling loops elsewhere without justification.
- **[enforced] Domain types mirror the loop-task daemon** in
  `src/renderer/src/types.ts` (source of truth: loop-task `src/types.ts` +
  `GET /api/openapi.json`). Do not invent fields; keep the subset in sync.
- **[target] Organize renderer code with Feature-Sliced Design** (layers:
  `app` ā†’ `pages` ā†’ `widgets` ā†’ `features` ā†’ `entities` ā†’ `shared`). Imports may
  only point to layers **below** the current one; no upward or sibling
  cross-imports. See `@feature-sliced-design`.
- **[target] Use catalogued patterns from patterns.dev** (design, rendering, and
  performance patterns) rather than ad-hoc solutions. See
  `@react-nextjs-patterns` and https://www.patterns.dev/ai/skills/catalog/.
- **[target] Wire services with dependency injection** via `@inversify-hooks`
  rather than module-level singletons, so units are testable and swappable.

## Naming Conventions

- **[enforced] React component files are PascalCase `*.tsx`** (e.g. `LogViewer.tsx`)
  and export a same-named component; helpers/hooks/modules are lowerCamelCase
  `*.ts` (e.g. `format.ts`, `store.ts`).
- **[enforced] Case must match imports exactly** ā€” `forceConsistentCasingInFileNames`
  is on. A wrong-cased import will fail on case-sensitive systems.
- **[enforced] Versioned persistence keys:** localStorage keys carry a `.vN`
  suffix (`lta.instances.v1`, `lta.selectedInstance.v1`). Bump the version when
  the stored shape changes.
- **[target] FSD slice/segment naming** ā€” group by feature slice with
  `ui/ model/ api/ lib/` segments; keep names domain-oriented.

## Code Style

- **[enforced] Strict TypeScript, no `any`.** `strict`, `noUnusedLocals`, and
  `noUnusedParameters` are enabled ā€” remove unused imports/vars/params or the
  build fails. Type the IPC boundary explicitly.
- **[enforced] ESM only.** `package.json` is `"type": "module"`; use explicit
  extensions on relative imports where the tsconfig requires it (main/preload
  import shared types as `../shared/ipc.js`).
- **[enforced] Prefer function components + hooks** (React 19). No class
  components.
- **[enforced] No inline hardcoded colors in components** ā€” use the CSS custom
  properties / design tokens defined in `DESIGN.md` / `theme.css`
  (`var(--accent)`, `STATUS_COLORS`, etc.). Loop status colors must stay matched
  to the loop-task TUI palette.
- **[target] No magic strings for user-facing text.** All UI copy goes through
  i18n keys via `@web-i18n-next-intl` (next-intl). English is the only locale
  for now, but strings still MUST be keyed ā€” never hardcode display text in JSX.
- **[target] Styling via Tailwind + shadcn** (`@tailwind-shadcn`) for new UI;
  the current plain-CSS token system (`theme.css`) is the reference palette to
  carry over, not to extend ad-hoc.
- **[target] Follow for structure, error
  handling, and readability; follow `@typescript-eslint` lint rules.

## Data & State

- **[enforced] The app owns no backend store.** Authoritative data lives in the
  loop-task daemons; treat daemon responses as the source of truth.
- **[enforced] Local persistence is limited to** versioned `localStorage`
  (renderer, via `store.ts`) and `window-bounds.json` in the Electron `userData`
  dir (main). Version/migrate keys when their shape changes.
- **[enforced] Unwrap the loop-task envelope** (`{ ok, data }` /
  `{ ok, error: { message } }`) into the app's `ApiResponse` in the main process;
  the renderer consumes the unwrapped shape only.
- **[target] Manage cross-cutting state through DI-provided services**
  (`@inversify-hooks`); keep view state local to components.

## Security

- **[enforced] Never weaken the renderer sandbox:** keep
  `contextIsolation: true` and `nodeIntegration: false`. Do not expose Node,
  `ipcRenderer`, or filesystem access to the renderer ā€” only the typed
  `window.api` bridge.
- **[enforced] Validate instance base URLs** as `http:`/`https:` in the main
  process before any request (see `isAllowedBaseUrl`).
- **[enforced] External links open in the system browser** via
  `shell.openExternal` (`setWindowOpenHandler` denies in-app navigation). Never
  navigate the app window to arbitrary URLs.
- **[enforced] No secrets in the repo.** The daemon API is unauthenticated/local;
  do not add tokens or credentials to source or persisted config.
